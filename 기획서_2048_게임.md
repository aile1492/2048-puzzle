## 1. 게임 개요

### 1.1 컨셉

클래식 2048 넘버 퍼즐의 정수를 살리면서, 다양한 보드 크기·게임 모드·테마로 차별화한 캐주얼 퍼즐 게임. 광고 기반 F2P 모델로 수익화한다.

### 1.2 핵심 가치

| 가치 | 설명 |
|------|------|
| **즉시 플레이** | 설치 후 3초 내 게임 시작 가능, 튜토리얼 최소화 |
| **원핸드 조작** | 한 손 스와이프만으로 완전한 플레이 |
| **무한 리플레이** | 레벨 디자인 불필요, 100% 절차적 생성 |
| **짧은 세션** | 1회 플레이 2~5분, 빈틈 시간 활용 |

### 1.3 타겟 유저

| 구분 | 내용 |
|------|------|
| **1차 타겟** | 캐주얼 퍼즐 유저 (15~45세, 성별 무관) |
| **2차 타겟** | 두뇌 트레이닝/수학 교육 목적 유저 |
| **이용 상황** | 출퇴근, 대기 시간, 취침 전, 휴식 시간 |

## 2. 핵심 게임플레이

### 2.1 기본 규칙

| 항목 | 규칙 |
|------|------|
| **보드** | NxN 격자 (기본 4x4) |
| **시작** | 빈 보드에 타일 2개 랜덤 배치 |
| **새 타일** | 매 이동 후 빈 칸에 1개 생성. **90% 확률 = 2, 10% 확률 = 4** |
| **이동** | 상/하/좌/우 스와이프 → 모든 타일이 해당 방향 끝까지 슬라이드 |
| **병합** | 같은 숫자 타일 충돌 시 합산 (2+2=4, 4+4=8...) |
| **이중 병합 금지** | 한 번의 이동에서 이미 병합된 타일은 다시 병합 불가 |
| **점수** | 병합 시 생성된 타일의 값 누적 합산 |
| **승리** | 2048 타일 생성 (이후 계속 플레이 가능) |
| **패배** | 빈 칸 없음 AND 인접한 같은 값 타일 없음 |

### 2.2 이중 병합 금지 예시

```
이동 전 (← 왼쪽 스와이프):  [2] [2] [2] [2]
결과:                       [4] [4] [ ] [ ]
→ 2+2=4 두 쌍이 각각 병합, 4+4=8로 연쇄 병합되지 않음
```

### 2.3 핵심 알고리즘 (의사코드)

#### 한 줄 처리: Compress → Merge → Compress

```gdscript
func process_row(row: Array) -> Dictionary:
    var score := 0

    # Step 1: Compress (빈칸 제거, 이동 방향으로 밀기)
    var compressed := row.filter(func(v): return v != 0)

    # Step 2: Merge (인접한 같은 값 병합)
    for i in range(compressed.size() - 1):
        if compressed[i] == compressed[i + 1]:
            compressed[i] *= 2
            score += compressed[i]
            compressed[i + 1] = 0

    # Step 3: Compress again (병합 후 빈칸 다시 제거)
    compressed = compressed.filter(func(v): return v != 0)

    # 빈칸 패딩
    while compressed.size() < row.size():
        compressed.append(0)

    return {"row": compressed, "score": score}
```

#### 4방향 처리 전략

| 방향 | 처리 방법 |
|------|----------|
| **← 왼쪽** | 각 행을 그대로 `process_row` |
| **→ 오른쪽** | 각 행을 뒤집고 → `process_row` → 다시 뒤집기 |
| **↑ 위** | 열을 행으로 변환 → `process_row` → 다시 열로 |
| **↓ 아래** | 열을 행으로 변환 → 뒤집고 → `process_row` → 뒤집고 → 다시 열로 |

#### 게임오버 판정

```gdscript
func is_game_over() -> bool:
    if has_empty_cell():
        return false
    for r in range(grid_size):
        for c in range(grid_size):
            if c < grid_size - 1 and grid[r][c] == grid[r][c + 1]:
                return false
            if r < grid_size - 1 and grid[r][c] == grid[r + 1][c]:
                return false
    return true
```

#### 새 타일 생성

```gdscript
func spawn_tile():
    var empty_cells := get_all_empty_cells()
    if empty_cells.is_empty():
        return
    var pos = empty_cells[randi() % empty_cells.size()]
    grid[pos.x][pos.y] = 2 if randf() < 0.9 else 4
```

## 6. 애니메이션 & 사운드

### 6.1 애니메이션 사양 (리서치 수치 반영)

> **리서치**: 슬라이드 100~150ms가 반응성의 핵심. 200ms 넘기면 "느리다" 피드백.
> 모든 타일이 **동시에** 움직여야 함 (순차 이동 절대 금지).
> 병합 시 **1.1~1.2배 바운스** 효과가 표준.

| 애니메이션 | 시간 | 이징 | 설명 | 리서치 근거 |
|-----------|------|------|------|-----------|
| **타일 슬라이딩** | 100~150ms | ease-out | 현재 위치 → 목표 위치 (동시) | 리서치: 100~150ms, 200ms 초과 시 불만 |
| **타일 생성** | 80~120ms | ease-out | scale 0 → 1 팝인 | 슬라이드 완료 후에 등장 |
| **타일 병합** | 80~120ms | ease-in-out | scale 1 → 1.15 → 1 바운스 | 리서치: 1.1~1.2배 |
| **점수 팝업** | 400~600ms | ease-out | +값 30~50px 위로 떠오르며 페이드아웃 | 리서치 표준 |
| **점수판 카운트업** | 300ms | ease-out | 숫자 롤링 애니메이션 | 리서치: 상단 점수도 카운트업 |
| **게임오버** | 500ms | ease-in | 격자 위에 반투명 오버레이 페이드인 | 고평점: 보드 완전히 가리지 않음 |
| **승리 연출** | 800ms | bounce | 2048 타일 골든 글로우 + 확대 효과 | 파티클 효과 추가 가능 |
| **화면 전환** | 200ms | ease-in-out | 페이드 또는 슬라이드 | 2초 내 앱 시작 목표 |

### 6.2 애니메이션 핵심 원칙

> **입력은 애니메이션을 기다리지 않는다.** (리서치 확인된 가장 중요한 원칙)
>
> 유저가 빠르게 연타해도 즉시 반영. 애니메이션은 비동기로 따라감.
> 애니메이션 중 입력을 **큐에 저장** (입력 버퍼링) → 완료 후 즉시 실행.
> 이것이 반응성의 핵심이며, 원본 2048의 쾌적함의 비결.
> **고평점 게임은 60fps를 안정적으로 유지.** 프로파일링 필수.

### 6.3 사운드 디자인

> **리서치**: 숫자가 커질수록 음높이가 올라가는 효과음 (음악적 스케일) 패턴이 호평.
> BGM은 대부분 없음 (퍼즐 집중 방해 금지). 모든 소리 개별 ON/OFF 가능해야 함.

| 사운드 | 트리거 | 특성 | 길이 |
|--------|--------|------|------|
| **타일 이동** | 스와이프 실행 | 부드러운 "쉭" 슬라이드 | ~100ms |
| **타일 병합 (소)** | 2~64 생성 | 낮은 톤 "톡" | ~150ms |
| **타일 병합 (중)** | 128~512 생성 | 중간 톤 "팝" | ~150ms |
| **타일 병합 (대)** | 1024+ 생성 | 높은 톤 "딩" (음악적 스케일) | ~200ms |
| **게임오버** | 이동 불가 | 하강 톤 | ~500ms |
| **승리** | 2048 달성 | 축하 팡파레 | ~1000ms |
| **버튼 클릭** | UI 버튼 탭 | 가벼운 클릭 | ~50ms |
| **Undo** | 되돌리기 | 되감기 효과음 | ~150ms |

### 6.4 햅틱 피드백 (리서치 추가)

> **리서치**: 고평점 게임은 단계별 햅틱으로 촉각 피드백 제공. 반드시 설정에서 끌 수 있어야 함.

| 트리거 | 강도 | 패턴 |
|--------|------|------|
| 스와이프 실행 | 약함 (Light) | 짧은 진동 |
| 일반 병합 | 중간 (Medium) | 짧은 진동 |
| 대형 병합 (128+) | 강함 (Heavy) | 짧은 진동 |
| 2048 달성 | 성공 패턴 | 긴 성공 진동 |
| 게임오버 | 실패 패턴 | 하강 진동 |

### 6.4 사운드 에셋 조달

- **무료 소스**: freesound.org, mixkit.co (CC0 라이선스)
- **필요 파일 수**: 약 7~10개
- **포맷**: OGG Vorbis (Godot 권장)
- **총 용량**: < 500KB 예상

### 8.2 핵심 클래스 설계

#### GridLogic (순수 데이터 로직, 비주얼 분리)

```
class GridLogic:
    Properties:
        - grid: Array[Array[int]]      # NxN 2D 배열
        - grid_size: int               # 3~6
        - score: int                   # 현재 점수
        - best_score: int              # 최고 점수
        - move_count: int              # 이동 횟수
        - highest_tile: int            # 최고 타일 값
        - history: Array[Dictionary]   # Undo용 이전 상태 스택

    Methods:
        - move(direction: Direction) -> MoveResult
        - spawn_tile() -> Vector2i
        - is_game_over() -> bool
        - has_won() -> bool
        - undo() -> bool
        - get_empty_cells() -> Array[Vector2i]
        - reset()
        - to_dict() -> Dictionary      # 직렬화
        - from_dict(data: Dictionary)   # 역직렬화

    Signals:
        - tiles_moved(movements: Array[TileMovement])
        - tile_merged(pos: Vector2i, value: int)
        - tile_spawned(pos: Vector2i, value: int)
        - score_changed(new_score: int)
        - game_over()
        - game_won()
```

#### TileMovement (이동 정보 구조체)

```
class TileMovement:
    - from_pos: Vector2i
    - to_pos: Vector2i
    - value: int
    - merged: bool
    - merge_value: int    # 병합 후 값 (병합 시만)
```

### 8.4 로직/비주얼 분리 원칙

```
[GridLogic] ──signal──> [GridBoard (비주얼)]
     ↑                       ↑
  순수 데이터              Tween 애니메이션
  Array 조작              타일 노드 이동/생성/삭제
  테스트 가능              화면에 표시
```

- `GridLogic`은 Node를 상속하지 않는 순수 GDScript 클래스
- `GridBoard`는 Control 노드로 GridLogic의 시그널을 받아 비주얼 처리
- 이렇게 분리하면 로직 유닛 테스트가 용이하고, 비주얼 변경이 로직에 영향 없음

### 13.1 4.5+ 별점 체크리스트 (리서치 기반)

> **Must-Have** (이것 없이는 4.0 미만)

- [ ] 게임 중간에 전면 광고 절대 넣지 않음 (게임오버/자연스러운 전환점에서만)
- [ ] 배너 광고가 게임 보드에 절대 겹치지 않음
- [ ] 60fps 애니메이션, 입력 지연 없음
- [ ] 앱 시작 2초 이내
- [ ] 자동 저장 완벽 (앱 강제 종료, 전화 수신, 배터리 방전 모든 상황)
- [ ] Undo 기능 (무료 3~5회)
- [ ] 일관된 디자인 시스템, 부드러운 애니메이션
- [ ] 오프라인 완전 지원

> **Nice-to-Have** (상위 1% 차별화)

- [ ] 다크 모드 + 테마 커스터마이징
- [ ] 통계 대시보드
- [ ] 광고 제거 IAP ($2.99)
- [ ] 데일리 챌린지 + 연속 출석
- [ ] 불필요한 권한 요청 없음 (위치, 연락처 등 절대 금지)

> **하지 말 것** (저평점 게임 공통 실수)

- 매 게임 사이 전면 광고
- 강제 튜토리얼 5단계+
- 불필요한 권한 요청
