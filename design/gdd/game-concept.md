# 게임 컨셉 문서 — 2048 퍼즐 게임

> **문서 버전**: 1.0
> **작성일**: 2026-03-23
> **작성자**: Game Designer (역공학 기반 문서화)
> **엔진**: Godot 4.6.1 / GDScript / 모바일 세로 1080×1920

---

## 1. 개요

2048은 4×4 격자(기본값) 위에서 타일을 상하좌우로 밀어 같은 숫자를 합산하며 2048 타일 도달을 목표로 하는 싱글플레이어 숫자 퍼즐 게임이다. 플레이어는 스와이프 한 번으로 격자 전체의 타일을 동시에 이동시키며, 합산이 발생할 때마다 점수를 획득한다. 2048에 도달하면 승리 팝업이 표시되고 계속 플레이를 선택하면 더 높은 타일을 향해 진행할 수 있다. Classic, Zen, Time Attack, Daily Challenge, Drop 다섯 가지 모드로 구성되어 있으며, 코인 경제 및 파워업(Hammer, Shuffle, Bomb) 시스템이 플레이의 깊이를 더한다. 매일 변경되는 시드 기반 Daily Challenge와 타일이 위에서 떨어지는 Drop 모드는 핵심 공식에 변형을 가해 반복 플레이 동기를 제공한다.

## 2. 플레이어 판타지

### 목표 감성 (MDA 프레임워크 — Aesthetics 계층)

| 우선순위 | 감성 유형 | 구체적 경험 |
|---------|----------|------------|
| 1순위 | **Challenge (도전)** | "한 번 더 하면 더 높은 타일에 도달할 수 있다"는 숙련 성장감 |
| 2순위 | **Submission (몰입)** | 지하철이나 잠들기 전, 생각을 비우는 반복적 행동의 리듬감 |
| 3순위 | **Discovery (발견)** | Drop 모드와 연쇄 병합에서 예상치 못한 콤보를 발견하는 쾌감 |
| 4순위 | **Achievement (성취)** | 일일 챌린지 목표 달성, 신기록 갱신, 최고 타일 업데이트 |

### Self-Determination Theory 매핑

- **자율성(Autonomy)**: 어느 방향으로 밀지 매 순간 선택. 파워업 사용 여부와 타이밍도 플레이어 결정.
- **유능감(Competence)**: 이동 결과가 즉각 시각 피드백으로 반환됨. 점수 증가, 타일 색 변화, 플로터 텍스트가 성장을 확인시켜 줌.
- **관계감(Relatedness)**: Daily Challenge 공유 시드로 같은 날 전 세계 플레이어가 동일한 보드를 경험. 리더보드(Rank)는 비교 동기 제공.

### 플로우 채널 설계

- **진입 장벽**: 첫 실행 시 파워업 스타터 팩(Hammer×3, Shuffle×1, Bomb×1) 지급으로 초기 장벽 완화.
- **난이도 곡선**: Classic 모드는 스스로 정체고원(plateau)을 형성하는 자기 조절형. Drop 모드는 타일이 쌓이면서 점진적 긴박감 상승.
- **실패 비용**: 즉각 재시작 가능, 이전 게임 상태 저장(current_game 섹션), Continue 1회 한정 제공으로 손실 회피 심리를 완충.

## 3. 상세 규칙

### 3-1. 공통 핵심 규칙 (Classic / Zen / Time Attack / Daily Challenge)

#### 격자 및 타일

- 격자는 N×N 크기로 구성되며 기본값은 4×4.
- 각 셀은 2의 거듭제곱 양의 정수 값 또는 0(빈 셀)을 가진다.
- 게임 시작 시 빈 격자에 타일 2개가 랜덤 스폰된다.

#### 이동 규칙

1. 플레이어가 상/하/좌/우 방향을 입력한다.
2. 해당 방향으로 모든 타일이 슬라이드한다. 타일은 빈 셀을 건너뛰어 격자 끝 또는 다른 타일에 닿을 때까지 이동한다.
3. 이동 방향을 기준으로 먼저 도달하는 쌍이 병합 우선권을 갖는다.
4. **병합 조건**: 동일한 값을 가진 두 타일이 인접하면 하나로 합산된다 (값 = 기존값 × 2).
5. **병합 제한**: 한 번 병합된 타일은 같은 이동 내에서 다시 병합될 수 없다.
6. 실제 위치 변화 또는 병합이 하나라도 발생하면 유효한 이동으로 처리된다.
7. 유효한 이동 후 새 타일 1개가 빈 셀 중 하나에 랜덤 스폰된다.
8. 아무것도 움직이지 않으면 이동 횟수가 증가하지 않으며 타일도 스폰되지 않는다.

#### 점수 계산

- 병합이 발생할 때마다 병합된 값(기존값 × 2)이 점수에 즉시 합산된다.
- 타일 이동만으로는 점수가 발생하지 않는다.

#### 타일 스폰

- 유효한 이동 후 빈 셀 중 임의의 위치 하나에 타일이 스폰된다.
- 값은 2 (확률 90%) 또는 4 (확률 10%).

#### 승리 조건

- 격자 내 최고 타일 값이 2048 이상이 되면 `game_won` 신호가 발행된다.
- 승리 팝업에서 "계속 플레이" 선택 시 `_continue_after_win = true`로 전환되어 더 높은 타일을 향해 계속 플레이 가능하다.
- 계속 플레이 중에는 2048 승리 조건이 재발동되지 않는다.

#### 패배 조건

- 모든 셀이 채워지고, 수평 및 수직으로 인접한 어떤 두 셀도 같은 값을 갖지 않을 때 게임 오버.
- 게임 오버 후 Continue 팝업이 표시되며 최초 1회에 한해 계속 진행이 가능하다 (`continue_used` 플래그).
- Continue 이후 다시 게임 오버가 되면 재시작만 가능하다.

#### 실행 취소 (Undo)

- Classic / Time Attack / Daily Challenge: 이동 하나씩 최대 3회 되돌리기 가능.
- 실행 취소는 히스토리 스택에서 직전 상태(grid, score, move_count, highest_tile)를 복원한다.
- 히스토리 스택 최대 깊이: 20.

### 3-2. 모드별 규칙

#### Classic 모드

- 기본 규칙을 그대로 따른다.
- 격자 크기: 4×4 (기본). 모드 선택 화면에서 다른 크기 선택 가능 (BOARD_SIZES 모드 참조).
- 제한 시간 없음, 이동 횟수 제한 없음.
- 최고 점수는 격자 크기별로 별도 저장된다: `best_score_4x4`, `best_score_5x5` 등.

#### Zen 모드

- Classic 규칙 동일.
- **실행 취소 무제한**: `request_undo()`가 항상 true를 반환한다.
- **파워업 비활성화**: 파워업 바가 숨겨진다.
- **타이머 미표시**: 시간 박스가 숨겨진다.
- 목적: 점수나 제한 없이 집중과 이완을 위한 플레이.

#### Time Attack 모드

- Classic 규칙에 카운트다운 타이머 추가.
- **제한 시간**: 120초 (2분).
- 타이머가 0에 도달하면 게임 오버가 즉시 트리거된다.
- 남은 시간이 10초 미만일 때 타이머 텍스트가 빨간색으로 변하고 펄스 애니메이션이 시작된다.
- 실행 취소: 최대 3회.

#### Daily Challenge 모드

- Classic 규칙에 날짜 기반 고정 시드와 목표 점수가 추가된다.
- **시드**: 오늘 날짜로부터 `YYYYMMDD` 정수 값이 RNG 시드로 사용된다. 같은 날 모든 플레이어가 동일한 초기 상태를 경험한다.
- **목표 점수**: 요일별 차등 적용 (월요일이 가장 낮고, 일요일이 가장 높음).
- 헤더의 "BEST" 박스가 "TARGET"으로 대체되어 목표 점수를 표시한다.
- 게임 오버 시 일반 게임오버 팝업 대신 `daily_result_popup`이 표시된다. 최종 점수가 목표 이상이면 성공.
- 실행 취소: 최대 3회.

#### Drop 모드

Drop 모드는 GridLogic 대신 DropLogic을 사용하는 독립적인 변형 모드이다.

**격자 구성**: 5열 × 8행. 행 0 = 최상단.

**기본 플로우**:
1. 화면 상단에 다음에 드롭할 타일의 값이 예고된다.
2. 플레이어가 열을 탭하면 해당 열의 최하단 빈 행에 타일이 착지한다.
3. 착지 후 같은 값을 가진 인접 타일(상하좌우)들이 연결된 그룹을 이루면 자동 병합된다.
4. 병합 후 중력이 적용되어 위에 있던 타일들이 아래로 내려온다.
5. 중력 적용 후 다시 병합을 검사하며, 더 이상 병합이 없을 때까지 반복한다. 이것이 **연쇄 콤보(chain)**이다.
6. 새 타일 값이 생성된다.

**병합 규칙 (Drop 모드 고유)**:
- N개의 같은 값 타일이 하나의 연결 그룹을 이루면: 결과 값 = 초기값 × 2^(ceil(log2(N)))의 단순화 버전으로, N개를 짝지어 순차 병합한다.
  - 예: 4개의 타일 → 2쌍 병합 → 값이 2배, 다시 2배 = 원래 값 × 4
- 착지 위치는 타일들 중 가장 낮은 행(최하단)에 결과 타일이 유지된다.

**연쇄 콤보 점수**: 각 연쇄 단계에서 병합 결과 값에 연쇄 횟수(chain_count)를 곱하여 점수를 계산한다.

**패배 조건**: 최상단 행(행 0)의 어느 열이라도 타일이 있으면 게임 오버.

**승리 조건**: 최고 타일이 2048 이상.

### 3-3. 파워업 시스템

파워업은 게임 중 격자 상태를 직접 조작하는 특수 도구이다. Zen 모드에서는 사용 불가.

#### Hammer (망치)

- **효과**: 선택한 셀 하나의 타일을 제거한다.
- **사용법**: Hammer 버튼 탭 → 선택 모드 진입 → 제거할 타일 탭 → 즉시 실행.
- **빈 셀 보호**: 빈 셀을 선택하면 파워업이 소비되지 않는다.
- **구현**: `GridLogic.remove_tile(pos)` 호출. 히스토리에 상태 저장(실행 취소 가능).
- **코인 구매가**: 100 코인.

#### Shuffle (셔플)

- **효과**: 격자 내 모든 타일의 위치를 Fisher-Yates 알고리즘으로 랜덤하게 재배치한다. 타일 값은 유지되고 위치만 변경된다.
- **사용법**: Shuffle 버튼 탭 → 즉시 실행 (선택 모드 없음).
- **구현**: `GridLogic.shuffle_tiles()` 호출. 히스토리에 상태 저장.
- **코인 구매가**: 150 코인.

#### Bomb (폭탄)

- **효과**: 선택한 셀과 상하좌우 4방향 인접 셀(총 최대 5개)의 타일을 모두 제거한다. 경계 밖 셀은 무시된다.
- **사용법**: Bomb 버튼 탭 → 선택 모드 진입 → 타일 탭 → 즉시 실행.
- **빈 셀 보호**: 선택한 셀이 빈 셀이면 파워업이 소비되지 않는다. 인접 빈 셀은 그냥 무시된다.
- **구현**: `GridLogic.remove_area(pos)` 호출. 히스토리에 상태 저장.
- **코인 구매가**: 200 코인.

#### 파워업 획득 경로

1. **스타터 팩**: 최초 실행 시 Hammer×3, Shuffle×1, Bomb×1 자동 지급.
2. **코인 구매**: 각 파워업을 코인으로 구매.
3. **보상형 광고 시청**: 하루 파워업 종류별 최대 3회까지 광고 시청으로 획득.

#### 선택 모드 흐름

Hammer / Bomb 사용 시:
1. 파워업 바의 버튼 탭.
2. 스와이프 입력 핸들러가 비활성화된다 (`_input_handler.set_process_input(false)`).
3. 선택 힌트 레이블이 표시된다.
4. 플레이어가 격자의 타일을 탭.
5. 파워업 실행, 선택 모드 종료, 입력 핸들러 재활성화.

## 4. 공식

### 4-1. 점수 공식 (Classic / Zen / Time Attack / Daily Challenge)

```
점수 증가량 = 병합된_타일_값
```

- **예시**: 값 4인 두 타일이 병합되면 +8 점. 값 512인 두 타일이 병합되면 +1024 점.
- 한 번의 스와이프에서 여러 병합이 동시에 발생하면 모든 병합 증가량이 합산된다.
- 공식 출처: `grid_logic.gd` `_merge_line()` 함수에서 `score_gained += info["merged_value"]`

### 4-2. 점수 공식 (Drop 모드 — 연쇄 콤보)

```
라운드_증가량 = sum(병합된_타일_값 × chain_count, 해당_연쇄_단계의_모든_병합)
총_점수_증가량 = sum(모든_연쇄_단계의_라운드_증가량)
```

- **chain_count**: 하나의 드롭 이벤트 내에서 중력-병합 사이클이 반복된 횟수.
- **예시**: 타일 하나가 착지 → 1차 병합(chain=1)에서 4+4=8 타일 1개 (+8×1=8) → 중력 → 8+8=16 병합(chain=2)이 발생 (+16×2=32) → 총 40점.
- 공식 출처: `drop_logic.gd` `drop_tile()` 함수.

### 4-3. 타일 스폰 확률 (Classic 계열)

```
P(값=2) = 0.90  (90%)
P(값=4) = 0.10  (10%)
```

- 위치는 빈 셀 목록에서 균일 분포 랜덤 선택.
- 공식 출처: `grid_logic.gd` `spawn_tile()`.

### 4-4. 다음 타일 생성 확률 (Drop 모드)

```
P(값=2) = 0.70  (70%)
P(값=4) = 0.25  (25%)
P(값=8) = 0.05  (5%)
```

- 공식 출처: `drop_logic.gd` `_generate_next()`.

### 4-5. 일일 챌린지 시드

```
seed = YYYY × 10000 + MM × 100 + DD
```

- **예시**: 2026년 3월 23일 → seed = 20260323.
- 동일한 seed로 초기화된 RNG는 항상 동일한 초기 2개 타일을 스폰한다.
- 공식 출처: `daily_seed.gd` `get_today_seed()`.

### 4-6. 일일 챌린지 목표 점수

```
요일 인덱스: 일=0, 월=1, 화=2, 수=3, 목=4, 금=5, 토=6
목표 점수 배열: [5000, 4000, 4500, 5000, 5500, 6000, 7000]
목표 점수 = 목표_점수_배열[요일_인덱스]
```

- **예시**: 월요일(weekday=1) → 목표 4000점 / 일요일(weekday=0) → 목표 5000점.
- **주의**: Godot `Time.get_date_dict_from_system()`의 weekday는 일요일=0, 토요일=6.
- 공식 출처: `daily_seed.gd` `get_daily_target_score()`.

### 4-7. Drop 모드 병합 그룹 결과 값

```
N = 연결된_같은_값_타일_개수
병합_횟수 = ceil(log2(N))  /* 짝수 올림 */
결과_값 = 초기_값 × 2^(병합_횟수)
```

- **구현 방식**: 코드는 while 루프로 `merge_count = (merge_count + 1) / 2` (정수 나눗셈)을 반복하며 result_value를 2배씩 증가시킨다.
- **예시**: 초기값 2, N=3 → 첫 루프: 3→2 / 두 번째 루프: 2→1 (종료) → 2배 × 2 = 8 (2×2²). N=4 → 4→2→1 (루프 2회) → 2×4 = 8.
- 공식 출처: `drop_logic.gd` `_find_and_execute_merges()`.

### 4-8. 최고 점수 저장 키

```
best_score_키 = "best_score_" + grid_size + "x" + grid_size
예시: 4×4 → "best_score_4x4"
```

## 5. 엣지 케이스

### 5-1. 격자 관련

| 상황 | 처리 방식 |
|------|----------|
| 이동 후 빈 셀이 없어 타일 스폰 불가 | `spawn_tile()`이 `Vector2i(-1, -1)` 반환, 스폰 없이 진행 |
| 이동해도 아무것도 바뀌지 않음 | 히스토리 스택에서 직전 항목 제거(pop_back), 이동 횟수 불증가 |
| 최고 타일이 2048 초과 후 다시 2048 도달 | `_won = true`이므로 `game_won` 재발행 안됨 |
| 계속 플레이 중 게임 오버 | 일반 게임오버 팝업 표시, Continue 버튼은 `continue_used` 플래그로 차단 |
| 히스토리 스택 20개 초과 | `pop_front()`로 가장 오래된 항목 자동 삭제 |

### 5-2. 병합 관련

| 상황 | 처리 방식 |
|------|----------|
| 한 줄에 동일한 값이 3개 이상 연속 (예: 4 4 4) | 이동 방향 기준 앞쪽 두 개가 먼저 병합 → 뒤의 하나는 이동만 (결과: 8 4) |
| 한 줄에 동일한 값이 4개 (예: 4 4 4 4) | 앞 두 개 병합, 뒤 두 개 병합 (결과: 8 8) |
| 이미 병합된 타일에 같은 값이 도달 | 같은 이동 내에서 재병합 금지. 다음 이동에서만 병합 가능 |

### 5-3. Drop 모드 관련

| 상황 | 처리 방식 |
|------|----------|
| 선택한 열이 가득 참 | `drop_tile()` 반환값 `{"landed": false}`, 아무 효과 없음 |
| 착지 후 병합이 없음 | chain_count = 0, `chain_completed` 신호 미발행 |
| 병합 후 중력으로 타일이 이동하여 새 그룹 형성 | while 루프 내에서 재탐색, 연쇄 병합 처리 |
| 착지 즉시 최상단 행(row 0)이 채워짐 | 병합/중력 처리 후 `is_game_over()` 검사하여 게임 오버 트리거 |

### 5-4. 파워업 관련

| 상황 | 처리 방식 |
|------|----------|
| Hammer를 빈 셀에 사용 | `remove_tile()` false 반환, `PowerUpManager.use_powerup()` 미호출, 소비 안 됨 |
| Bomb를 격자 모서리에 사용 | 범위 밖 셀은 `remove_area()` 내 경계 검사로 무시됨 |
| 인벤토리 0개 상태에서 사용 시도 | `powerup_purchase_popup` 표시, 실제 사용 안 됨 |
| 광고 시청 한도(일 3회) 초과 후 시도 | `can_watch_ad()` false, 광고 요청 미발행 |
| 광고 시청 날짜 리셋 | 로딩 시 저장된 날짜와 오늘 날짜 비교, 다르면 ad_uses_today 전부 0으로 초기화 |
| Zen 모드에서 파워업 버튼 탭 | `_on_powerup_requested()`에서 Zen 모드 체크 후 조기 반환 |

### 5-5. 일일 챌린지 관련

| 상황 | 처리 방식 |
|------|----------|
| 같은 날 게임을 다시 시작 | 동일한 seed로 동일한 초기 보드 재생성 |
| Daily Challenge를 Continue 후 게임 오버 | `daily_result_popup` 표시, continue_used 플래그로 1회 제한 |
| 요일 인덱스가 배열 범위 외 | 기본값 5000 반환 |

### 5-6. 저장/복원 관련

| 상황 | 처리 방식 |
|------|----------|
| 앱 종료 후 재진입 | `current_game` 섹션에 저장된 to_dict() 상태로 복원 (`resume: true` 데이터 전달) |
| 복원 데이터가 없거나 grid 키 누락 | 새 게임 시작 |
| 게임 오버/승리 후 current_game 클리어 | `SaveManager.set_section("current_game", {})` 호출 |

## 6. 의존성

### 6-1. 시스템 의존성 다이어그램

```
GameManager ──────────────────► game_screen.gd (상태 구독)
     │
     └── GridLogic (core 2048 로직) ◄─── grid_board.gd (UI 래퍼)
     └── DropLogic (Drop 모드 로직) ◄─── drop_board.gd (UI 래퍼)

PowerUpManager ──────────────► GridLogic (remove_tile, remove_area, shuffle_tiles)
     │                        ► AdManager (rewarded_ad_completed 구독)
     └── CoinManager (spend_coins)

CoinManager ─────────────────► SaveManager (progress/coins)

DailySeed ───────────────────► game_screen.gd (seed 제공)
                              ► PowerUpManager (오늘 날짜 문자열 비교)

SaveManager ─────────────────► 모든 Autoload (전역 저장소)

ThemeManager ────────────────► game_screen.gd (theme_changed 신호)
                              ► TileColors (색상 팔레트 제공)

AnalyticsManager ────────────► game_screen.gd (게임 시작/종료 이벤트 로그)

AudioManager ────────────────► game_screen.gd (효과음 재생)

ScreenManager ───────────────► game_screen.gd (팝업 표시, 화면 전환)
```

### 6-2. 통합 계약

| 이 시스템이 제공하는 것 | 이 시스템이 요구하는 것 |
|----------------------|----------------------|
| GridLogic: grid 상태, score, move_count, highest_tile, 신호(tiles_moved, tile_spawned, score_changed, game_over, game_won) | RNG seed(옵션), grid_size |
| DropLogic: grid 상태, score, chain 신호, game_over/game_won | cols, rows 파라미터 |
| GameManager: current_state, current_mode, undo_remaining 상태 | 없음 (최상위 상태 관리자) |
| PowerUpManager: get_count, use_powerup, purchase_with_coins, can_watch_ad | CoinManager.spend_coins, AdManager.show_rewarded_ad, SaveManager, DailySeed |
| CoinManager: get_coins, add_coins, spend_coins | SaveManager |
| DailySeed: get_today_seed, get_today_string, get_daily_target_score, is_same_day | Godot Time API |

## 7. 조정 가능 값 (Tuning Knobs)

모든 값은 외부 데이터 파일(`assets/data/`) 또는 각 autoload 상수로 분리되어야 한다. 현재 코드에 하드코딩된 값과 그 의도 범위를 기록한다.

### Feel 노브 (순간 경험 조정)

| 값 | 현재값 | 위치 | 의도 범위 | 설명 |
|----|--------|------|----------|------|
| 점수 플로터 최소 표시 점수 | 4 | game_screen.gd | 2~16 | 너무 자주 표시되면 노이즈 |
| 점수 플로터 이동 시간 | 0.7초 | game_screen.gd | 0.4~1.0초 | 눈으로 따라갈 수 있는 속도 |
| 최고 점수 플래시 지속 시간 | 0.3+0.4+0.5초 | game_screen.gd | 전체 1.0~1.5초 | 지나치게 길면 방해 |
| 타임 펄스 스케일 | 1.15× | game_screen.gd | 1.05~1.3× | 긴박감 vs 가독성 |

### Curve 노브 (진행 곡선 조정)

| 값 | 현재값 | 위치 | 의도 범위 | 설명 |
|----|--------|------|----------|------|
| 스폰 타일 값=2 확률 (Classic) | 0.90 | grid_logic.gd | 0.80~0.95 | 낮을수록 난이도 하락 (4 타일 빈도 증가) |
| 스폰 타일 값=4 확률 (Classic) | 0.10 | grid_logic.gd | 0.05~0.20 | 위와 역수 |
| Drop 모드 값=2 확률 | 0.70 | drop_logic.gd | 0.55~0.80 | 낮을수록 고값 타일 빈도 증가 |
| Drop 모드 값=4 확률 | 0.25 | drop_logic.gd | 0.15~0.35 | - |
| Drop 모드 값=8 확률 | 0.05 | drop_logic.gd | 0.02~0.10 | 너무 높으면 난이도 급락 |
| 일일 챌린지 목표 점수 배열 | [5000,4000,4500,5000,5500,6000,7000] | daily_seed.gd | 각 요소 ±2000 | 주 중반 상승, 주말 피크 |

### Gate 노브 (진행 관문 조정)

| 값 | 현재값 | 위치 | 의도 범위 | 설명 |
|----|--------|------|----------|------|
| 실행 취소 최대 횟수 (Classic 등) | 3 | game_manager.gd | 1~5 | 3이 표준 |
| Time Attack 제한 시간 | 120초 | game_screen.gd | 60~180초 | 120초가 긴박감/여유의 균형점 |
| 긴박 타이머 경고 임계값 | 10초 | game_screen.gd | 5~20초 | 10초 미만에서 빨간색 |

| 광고 시청 일일 한도 (파워업별) | 3회 | powerup_manager.gd | 2~5회 | 수익화 vs 사용자 경험 |
| 파워업 스타터 팩 수량 | Hammer×3, Shuffle×1, Bomb×1 | powerup_manager.gd | 조정 가능 | 온보딩 파워업 체험 |

## 8. 수용 기준

### 8-1. 기능적 수용 기준

#### Core 2048 로직

- [ ] 상/하/좌/우 이동 후 모든 타일이 올바른 위치로 이동한다.
- [ ] 같은 값의 인접 타일이 1회 이동에서 최대 1회 병합된다 (이중 병합 금지).
- [ ] 병합 시 `score_changed` 신호가 병합된 값(합산 결과)과 함께 발행된다.
- [ ] 이동이 발생하지 않으면 히스토리 스택에 항목이 추가되지 않는다.
- [ ] 유효한 이동 후 정확히 1개의 타일이 빈 셀에 스폰된다.
- [ ] 최고 타일이 2048 이상이 되면 정확히 1번 `game_won` 신호가 발행된다.
- [ ] 빈 셀이 없고 인접한 같은 값 쌍이 없으면 `game_over` 신호가 발행된다.
- [ ] `undo()`가 직전 grid, score, move_count 상태를 정확히 복원한다.

#### Drop 모드 로직

- [ ] 열을 탭하면 해당 열의 최하단 빈 행에 타일이 착지한다.
- [ ] 착지 후 같은 값의 인접 그룹이 자동으로 병합된다.
- [ ] 병합 후 중력이 적용되어 위 타일이 아래로 이동한다.
- [ ] 연쇄 콤보 발생 시 `chain_completed` 신호가 올바른 chain_count와 함께 발행된다.
- [ ] 최상단 행이 채워지면 `game_over` 신호가 발행된다.

#### 파워업 로직

- [ ] Hammer 사용 시 선택한 셀의 타일만 제거된다.
- [ ] Bomb 사용 시 선택한 셀과 상하좌우 인접 셀(최대 5개)이 제거된다.
- [ ] Shuffle 사용 시 타일 값의 집합은 동일하고 위치만 재배치된다.
- [ ] 빈 셀 선택 시 Hammer/Bomb의 인벤토리가 소모되지 않는다.
- [ ] 인벤토리가 0일 때 `use_powerup()`이 false를 반환한다.
- [ ] 광고 일일 한도 초과 후 `can_watch_ad()`가 false를 반환한다.

#### 일일 챌린지 로직

- [ ] 같은 날짜를 입력하면 항상 동일한 seed 값이 반환된다.
- [ ] 동일한 seed로 초기화된 GridLogic은 동일한 2개의 초기 타일을 생성한다.
- [ ] 요일별 목표 점수가 올바른 값을 반환한다.
- [ ] 게임 오버 시 `daily_result_popup`이 표시되고 score >= target이면 "성공"으로 표기된다.

#### Time Attack 로직

- [ ] 타이머가 120초에서 시작하여 실시간으로 감소한다.
- [ ] 10초 미만 시 타이머 레이블이 빨간색으로 변하고 펄스 애니메이션이 시작된다.
- [ ] 타이머가 0에 도달하면 즉시 게임 오버가 트리거된다.

#### 경제 로직

- [ ] 코인 지출 시 잔액이 부족하면 `spend_coins()`가 false를 반환한다.
- [ ] 코인 거래가 저장되고 앱 재시작 후에도 유지된다.

### 8-2. 경험적 수용 기준 (플레이테스트 검증)

| 검증 항목 | 목표 기준 | 측정 방법 |

| 첫 게임에서 게임 오버까지 도달하는 시간 | 3~10분 | 세션 로그 play_time |
